[PR] Refaktor Laporan: Implementasi Grouping per Item dan Lokasi

Deskripsi Singkat

Pull Request ini melakukan refaktor pada sistem pelaporan (pembelian dan penjualan) dengan mengubah struktur tampilan dari tabel tunggal (flat table) menjadi struktur pengelompokan (grouping). Perubahan ini bertujuan untuk memberikan perspektif yang lebih mendalam mengenai riwayat transaksi untuk setiap entitas tertentu (Barang atau Lokasi).

Motivasi Perubahan

Sebelumnya, laporan hanya menampilkan daftar transaksi secara berurutan, sehingga sulit bagi pemilik toko untuk:

Melihat total kuantitas dan nilai belanja/penjualan untuk satu barang spesifik dalam satu periode.

Memantau fluktuasi harga beli/jual dari waktu ke waktu untuk barang yang sama.

Melacak performa stok atau omzet per cabang (lokasi) secara terpisah.

Perubahan Utama

1. Perubahan Logika Tampilan (EJS)

Implementasi Grouping: Menambahkan logika JavaScript di dalam file EJS untuk memproses data mentah menjadi objek terkelompok sebelum proses rendering.

POV Transaksi: Mengubah tampilan menjadi blok-blok tabel per entitas (Barang/Lokasi).

Subtotal Otomatis: Setiap blok sekarang memiliki baris <tfoot> yang menjumlahkan kolom Quantity dan Subtotal secara spesifik hanya untuk entitas tersebut.

2. Peningkatan Antarmuka (UI/UX)

Visual Branding: Menggunakan skema warna brand Grfyn (#C4683D) untuk header grup dan border.

Keterbacaan: Menambahkan zebra striping pada baris tabel agar data transaksi yang padat lebih mudah dibaca.

Optimalisasi Cetak: Menambahkan aturan CSS page-break-inside: avoid; untuk memastikan setiap blok laporan tidak terpotong saat dicetak ke PDF atau fisik.

Cara Pengetesan

Buka menu Laporan Pembelian per Barang.

Filter berdasarkan periode tertentu.

Pastikan data tampil berkelompok berdasarkan Nama Barang.

Periksa apakah nilai Total Qty dan Total Nilai di bawah tabel barang sudah sesuai dengan jumlah baris di atasnya.

Ulangi langkah yang sama untuk laporan Penjualan dan laporan per Lokasi.

Coba tekan tombol Cetak dan pastikan layout di jendela print preview tertata rapi (tidak ada header yang terpisah dari tabelnya).
